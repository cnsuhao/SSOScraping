netfilter.org?</h2></div></div></div>\n\n<p>\nnetfilter.org is home to the software of the packet filtering framework inside the <a xmlns=\"http://www.w3.org/1999/xhtml\" href=\"http://www.kernel.org/\">Linux</a> 2.4.x and later kernel series. Software commonly associated with netfilter.org is <a href=\"projects/iptables/index.html\" class=\"olink\">iptables</a>.\n</p>\n<p>\nSoftware inside this framework enables packet filtering, network address [and\nport] translation (NA[P]T) and other packet mangling.  It is the re-designed\nand heavily improved successor of the previous Linux 2.2.x <a xmlns=\"http://www.w3.org/1999/xhtml\" href=\"/ipchains\">ipchains</a> and Linux 2.0.x <a xmlns=\"http://www.w3.org/1999/xhtml\" href=\"http://www.xos.nl/resources/ipfwadm/\">ipfwadm</a> systems.\n</p>\n<p>\nnetfilter is a set of hooks inside the Linux kernel that allows kernel modules\nto register callback functions with the network stack.  A registered callback\nfunction is then called back for every packet that traverses the respective\nhook within the network stack.\n</p>\n<p>\niptables is a generic table structure for the definition of rulesets.  Each\nrule within an IP table consists of a number of classifiers (iptables\nmatches) and one connected action (iptables target).\n</p>\n<p>\nnetfilter, ip_tables, connection tracking (ip_conntrack, nf_conntrack) and\nthe NAT subsystem together build the major parts of the framework.\n</p>\n</div>\n            <div xmlns=\"\" class=\"section\"><div class=\"titlepage\"><div><div><h2 class=\"title\" style=\"clear: both\"><a name=\"features\" id=\"features\"></a>Main Features</h2></div></div></div>\n\n<div class=\"itemizedlist\"><ul class=\"itemizedlist\" style=\"list-style-type: bullet;
