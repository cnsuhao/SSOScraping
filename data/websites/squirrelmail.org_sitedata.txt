's/draft_folder=Drafts/draft_folder=INBOX.Drafts/g' {} \\;\n<br>\n&nbsp;&nbsp;&nbsp;find /var/lib/squirrelmail/data/ -name *.pref -exec sed --in-place 's/sent_folder=Sent/sent_folder=INBOX.Sent/g' {} \\;\n<br><br>\nOr, for database-based preferences:\n<br><br>\n&nbsp;&nbsp;&nbsp;UPDATE userprefs SET prefval = 'INBOX.Trash' WHERE prefkey = 'trash_folder' AND prefval = 'Trash';<br>\n&nbsp;&nbsp;&nbsp;UPDATE userprefs SET prefval = 'INBOX.Drafts' WHERE prefkey = 'draft_folder' AND prefval = 'Drafts';<br>\n&nbsp;&nbsp;&nbsp;UPDATE userprefs SET prefval = 'INBOX.Sent' WHERE prefkey = 'sent_folder' AND prefval = 'Sent';\n<br><br>\nMAKE SURE to back up your user preferences before doing any of the above!\n<br><br>\nThis release also addresses several security issues, including some harsh but hard to exploit XSS bugs, a general clickjacking vulnerability, and a small problem with message sanitizing.\n<br><br>\nIf only for the clickjacking protection, we recommend that users of previous versions of SquirrelMail upgrade at their earliest convenience.\n<br><br>\nFor more complete details, see the ReleaseNotes and ChangeLog files included in this release (in the
